# 异环驱动块伤害最大化计算器

本项目是一个本地静态 Web 工具，用于在 5x5 底板中搜索完全填满的驱动块组合，并自动搭配启用卡带后按期望伤害排序输出 Top N 方案。

## 当前规则

- 驱动块形状沿用 `NoroHime/NTE-DriveBlock-Solver` 的 2/3/4 格固定形状。
- 驱动块禁止旋转、禁止翻转。
- 求解器只接受完全填满底板的方案。
- 锁定驱动块表示该驱动块必须进入方案，位置由求解器搜索。
- 每个驱动块固定 4 个用户录入词条。
- 每个卡带包含 1 个主词条和 4 个用户录入词条。
- 用户录入驱动块/卡带时只选择稀有度和词条类型，数值由“数据表”自动派生。
- 驱动块基础攻击/生命按 `稀有度每格基础值 * 驱动块格子数` 计算。
- 驱动块词条按 `稀有度每格词条值 * 驱动块格子数` 计算。
- 卡带不区分 II/III/IV 型。
- 卡带主词条按 `稀有度主词条固定值` 计算。
- 卡带四个副词条按 `稀有度副词条固定值` 计算。
- 卡带副词条允许重复选择。
- 可添加多个必选形状；求解器会保证结果中至少包含这些形状。
- 可设置剩余空间搜索时的填充优先级：优先 II/III/IV 型。
- 卡带不占底板空间；每个完整驱动块方案会自动选择伤害最高的启用卡带。

## 伤害模型

当前实现基础伤害区、增伤区和暴击期望：

```text
期望伤害 = 基础伤害区 * 增伤区 * 暴击期望

基础伤害区 = Σ(伤害倍率 * 对应属性) + 额外伤害值
增伤区 = 1 + 通用伤害增强 + 属性伤害增强
暴击期望 = 1 + 暴击率 * 暴击伤害
```

心灵伤害增强、治疗加成、倾陷强度、环合强度目前只记录和展示，不参与当前伤害评分。

## 运行

当前实现不依赖 npm 包，可直接用 Node 启动静态服务：

```powershell
C:\Users\a\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe server.mjs --port 4173
```

打开：

```text
http://127.0.0.1:4173/
```
